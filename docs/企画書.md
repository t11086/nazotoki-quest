# ナゾトキクエスト 〜知識で解く事件簿〜 企画書

## 1. 企画概要

| 項目 | 内容 |
|---|---|
| アプリ名(仮) | ナゾトキクエスト 〜知識で解く事件簿〜 |
| ジャンル | 謎解きアドベンチャー型 知育アプリ |
| ターゲット | 小学6年〜中学3年の男女(11〜15歳) |
| プラットフォーム | まずは Web アプリ(スマホ・タブレット対応)。将来的にネイティブ化 |
| 一言コンセプト | 「勉強した知識が、事件を解くカギになる」 |

## 2. 背景・課題

- 小6〜中3は「勉強=やらされるもの」になりがちで、単純な暗記ドリル系アプリは飽きられやすい。
- 一方でこの年代は謎解き・脱出ゲーム・推理モノ(コナン、東大ナゾトレ等)への関心が非常に高い。
- 既存の学習アプリは「教科の知識を問う」ものが中心で、**知識を"使って"何かを解決する体験**が少ない。
- → 「知識はナゾを解く武器になる」という体験を通じて、学ぶ動機そのものを作る。

## 3. コンセプト

プレイヤーは「ナゾトキ探偵団」の新米探偵。
学園や街で起きる事件(盗まれた文化祭の看板、暗号の手紙、消えた実験データ…)を、
**教科の知識を手がかりに推理して解決する**ストーリー型謎解きアプリ。

- 単なる4択クイズではなく、「事件の証拠」として知識が登場する。
  - 例:暗号文がローマ字+英単語で書かれている(英語)
  - 例:犯行時刻をアリバイの移動速度から計算する(数学:速さ・距離・時間)
  - 例:「この足跡が乾いていない=雨が降った後」→ 天気図の読み取り(理科)
  - 例:古地図の地名から場所を特定する(社会:地理・歴史)
- 1事件 = 10〜15分で解けるボリューム。通学・すきま時間で1話完結。

## 4. ターゲット分析

| セグメント | 特徴 | 刺さるポイント |
|---|---|---|
| 小6 | 中学受験 or 中学準備期。ゲーム時間に親の目がある | 「勉強にもなる」と親が許可しやすい建て付け |
| 中1〜中2 | 部活・スマホ中心。勉強アプリは自発的には入れない | 推理・謎解きという入口。SNS的なランキング・称号 |
| 中3 | 受験期。すきま時間の息抜きを探している | 1話10分完結+出題範囲が入試基礎と対応 |
| 男女共通 | 性別で嗜好が分かれにくい題材が必要 | 「推理・謎解き」は男女差が小さいジャンル。キャラも等身大の探偵団 |

## 5. 主要機能

### 5.1 メイン:事件簿モード
- 章立てのストーリーに沿って事件を捜査。
- 「聞き込み → 証拠集め → 推理(謎解き)→ 解決編」の4フェーズ。
- 謎解きパートで教科知識を使う設問が出る。ヒントは3段階(ヒント使用でも進行は可能、報酬が減るだけ)。

### 5.2 デイリー捜査(1日1問)
- 毎日1問の小さな謎(暗号、計算トリック、雑学推理)。
- 連続ログインではなく**連続解決日数**でストリーク表示 → 学習習慣化。

### 5.3 探偵ランクと称号
- 事件解決で「推理ポイント」を獲得し、見習い → 助手 → 一人前 → 名探偵…とランクアップ。
- 教科別バッジ(数学の名手、暗号解読士 など)で得意分野を可視化。

### 5.4 復習ファイル(親・本人向け)
- 事件で使った知識を「捜査資料」として自動アーカイブ。
- どの教科・単元を使ったかが一覧で見える → 保護者への説明にもなる。

### 5.5 フレンド共闘(将来拡張)
- 難事件を友達と分担捜査(片方が暗号、片方が計算)。個人情報を出さないコード招待制。

## 6. 学習設計

- 出題範囲:小6〜中3の主要5教科(国・数・英・理・社)の基礎〜標準レベル。
- 学年設定に応じて謎の難度と使用単元を自動調整(小6には中学内容を出さない)。
- 「知識がないと解けない」ではなく「ヒント経由で知識を学べば解ける」設計にし、未習単元も入口にする。
- 間違えた・ヒントを多用した単元は、後の事件で形を変えて再登場(間隔反復)。

## 7. マネタイズ(案)

- 基本無料+月額サブスク(事件簿の章解放、ヒント無制限)。
- 買い切りの「特別事件パック」(長編1本)。
- 広告は入れない(保護者の信頼・年齢層への配慮を優先)。

## 8. 競合との差別化

| 競合 | 特徴 | 本企画の差別化 |
|---|---|---|
| 暗記ドリル系アプリ | 単元別の反復練習 | 知識を「使う」文脈がある。飽きにくい |
| 脱出ゲーム・謎解きアプリ | 娯楽特化、学習要素なし | 謎が教科カリキュラムと対応し、学習履歴が残る |
| 通信教育アプリ | 網羅的だが義務感が強い | 10分1話のストーリーで自発的に開きたくなる |

## 9. 成功指標(KPI)

- 継続率:7日後継続 40%、30日後継続 20%
- デイリー捜査の解答率:DAUの60%以上
- 1事件あたりの完了率:80%以上(途中離脱が多い謎は難度調整)

## 10. 開発ロードマップ(案)

| フェーズ | 内容 |
|---|---|
| Phase 1(プロトタイプ) | Webで事件1話+デイリー捜査を実装。謎解きUIの手応え検証 |
| Phase 2(MVP) | 第1章(5事件)、ランク・バッジ、復習ファイル |
| Phase 3 | 学年別難度調整、間隔反復、サブスク導入 |
| Phase 4 | フレンド共闘、ネイティブアプリ化 |

## 11. 技術構成(Phase 1 想定)

- フロント:React + TypeScript + Vite(スマホファーストのレスポンシブ)
- 状態・進行データ:まずは localStorage(アカウント不要で遊べる)
- 将来:バックエンド(認証・進行同期・ランキング)を追加
